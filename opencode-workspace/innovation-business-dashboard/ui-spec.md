# 创新业务看板 页面视觉规格

## 1. 输入来源

- **需求材料**：用户口述《创新业务看板需求》
- **requirement.md**：`./requirement.md`
- **reqdoc.md**：`./reqdoc.md`
- **目标项目**：华润雪花营销驾驶舱 H5（`marketing-cockpit-dashboard-h5`）
- **同类页面**：营销驾驶舱现有数据看板页面（销量快报、销量分析、库存分析等）
- **项目本地规则 / skills**：`snow-design-mobile/references/project-marketing-cockpit.md`

---

## 2. 页面骨架结论

- **阅读方式**：分屏看板型（Banner 导航按钮切换四大视角，每个视角内从上到下连续滚动展示分类汇总表格 → 组织层级页签 → 组织排名表格）
- **采用理由**：
  - 用户明确要求"Banner 需要导航按钮筛选整体、代工类、特色产品、桶啤"，四大视角为单选互斥
  - 每个视角内部结构一致（汇总表格 + 组织排名），适合连续滚动阅读
  - 与营销驾驶舱现有数据看板页面模式一致：Banner 区域 → 内容区 → 组织页签 → 数据表格
  - 区别于销量快报的"页面级 Tab + 子维度横滑"，本需求用"Banner 内导航按钮 + 子类分行列表"，层级更浅
- **用户已确认**：
  - 整体视角展示三大分类汇总对比 ✅
  - 子类一个表格分行列出 ✅
  - 组织按排名列表展示 ✅
  - 不要图表 ✅
- **按项目基线推断**：
  - Banner 区域采用雪花蓝渐变背景（参考销量快报，非 COS 图片 Hero）
  - 导航按钮采用胶囊按钮组，放在 Banner 内容流中
  - 组织层级页签采用分段器（参考销量快报"分事业部 / 分营销中心"页签）
  - 表格采用固定首列 + 横向滚动 + 进度条（参考销量快报数据表格）
  - Top3 蓝色圆形排名徽章（参考销量快报）

## 3. 页面整体结构

```
顶部导航栏（创新业务看板 + 返回）
    ↓
Banner 区域（雪花蓝渐变背景）
    ├── 看板标题：创新业务看板
    ├── 时间标签：2026年6月（可点击切换日/月/年）
    └── 导航按钮组：整体 | 代工类 | 特色产品 | 桶啤（单选互斥）
    ↓
分类汇总表格（动态行数，随视角变化）
    ├── 整体视角：代工类 / 特色产品 / 桶啤（3行 + 合计行）
    ├── 代工类视角：代加工 / 包销 / 定制（3行 + 合计行）
    ├── 特色产品视角：小品种 / 贸易产品 / 醴（3行 + 合计行）
    └── 桶啤视角：高档桶啤 / 普通桶啤（2行 + 合计行）
    ↓
组织层级页签（分段器：总部 | 区域 | 营销中心 | 工厂）
    ↓
组织排名表格（固定首列 + 横向滚动 + Top3徽章 + 进度条 + 正负着色）
    ↓
空状态 / 加载态 / 错误态（按需展示）
```

## 5. 模块视觉规格

### 5.1 顶部导航栏

- 高度：44px
- 背景：White（#FFFFFF）
- 标题：创新业务看板，17/Bold，Text-5（#0B0D0F），居中
- 返回按钮：左侧箭头，24×24，Text-4（#576675）
- 底部分隔线：1px Divider（#F0F0F0）

### 5.2 Banner 区域

- 背景：雪花蓝渐变 `linear-gradient(135deg, #0073E5 0%, #0052A3 100%)`
- 内边距：20px 16px 16px
- 内容（从上到下）：

#### 5.2.1 看板标题
- 文字：创新业务看板
- 字号：18/Medium
- 颜色：White（#FFFFFF）

#### 5.2.2 时间标签
- 文字：2026年6月（随口径变化：日=2026-06-26，月=2026年6月，年=2026年）
- 字号：12/Regular
- 颜色：White 80% 透明度（rgba(255,255,255,0.8)）
- 交互：点击弹出底部时间选择器（抽屉）
- 与标题间距：8px

#### 5.2.3 导航按钮组
- 位置：时间标签下方，间距 16px
- 布局：横向排列，4 个胶囊按钮等分或自适应宽度
- 按钮间距：8px
- 单个按钮样式：
  - 选中态：White（#FFFFFF）背景，Brand-6（#0073E5）文字，14/Medium
  - 未选中态：rgba(255,255,255,0.2) 背景，White 文字，14/Regular
  - 圆角：Radius-pill（999px）
  - 内边距：6px 16px
  - 最小高度：32px
- 交互：点击切换视角，单选互斥
- 4 个按钮：整体 | 代工类 | 特色产品 | 桶啤

### 5.3 分类汇总表格

- 背景：White（#FFFFFF）
- 圆角：8px
- 外边距：12px 16px
- 内边距：0（表格撑满卡片）
- 表头背景：Brand-1（#E6F2FF）
- 表头文字：11/Regular，Text-4（#576675）
- 数据行：
  - 行高：约 44px
  - 分隔线：1px Divider（#F0F0F0）
  - 分类/子类名称：13/Regular，Text-5（#0B0D0F），左对齐
  - 当期/同期/同比增减：13/Medium，Text-5，右对齐，DIN 字体
  - 同比增幅：13/Medium，右对齐，DIN 字体
  - 同比增减正数：Success-4（#2EB82E）
  - 同比增减负数：Danger-4（#F53D3D）
  - 同比增幅正数：Success-4 + ↑ 符号
  - 同比增幅负数：Danger-4 + ↓ 符号
  - 零增长：Text-3（#8A99A8），展示 0.0%
  - 同期为0：展示「NEW」标识，Brand-6 色
- 合计行：
  - 文字加粗（13/Bold）
  - 背景：Fill-1（#F7F7F8）
  - 不参与排名
- 单位行：表头或首行右侧标注「单位：千升(kL)」，11/Regular，Text-3

### 5.4 组织层级页签（分段器）

- 容器：圆角 4px，1px Brand-6（#0073E5）描边
- 高度：36px
- 外边距：0 16px 12px
- 4 个页签等分：总部 | 区域 | 营销中心 | 工厂
- 选中项：Brand-6 背景，White 文字，13/Medium
- 未选中项：透明背景，Brand-6 文字，13/Regular
- 权限不足项：置灰（Text-3 #8A99A8 文字 + 1px Divider 描边），不可点击
- 交互：点击切换组织层级，保留视角和时间状态
- 4 个页签在 375px 宽度下的适配：每个页签约 80px 宽度（(375-32-6)/4 ≈ 84px），字号 13px 可容纳"营销中心"4字

### 5.5 组织排名表格

- 背景：White（#FFFFFF）
- 圆角：8px
- 外边距：0 16px 16px
- 表头背景：Brand-1（#E6F2FF）
- 表头文字：11/Regular，Text-4（#576675）
- 固定首列：组织名称 + 排名徽章，宽度 96px
  - 排名徽章 Top3：Brand-6 圆形背景 20×20，White 数字 11/Bold
  - 排名 4+：Text-4（#576675）数字，13/Regular
  - 组织名称：13/Regular，Text-5（#0B0D0F）
  - 组织名超 6 字：独占一行，数据行空列占位对齐表头
- 横向滚动区：当期 / 同期 / 同比增减 / 同比增幅 / 增量贡献
  - 数据文字：13/Regular/Medium，Text-5，右对齐，DIN 字体
  - 正负着色规则同 5.3
- 增量贡献列：
  - 进度条：高 6px，Fill-2（#F1F1F4）背景，Brand-6 填充
  - 宽度 = 当期 / 当期最大值 × 100%
  - 进度条上方或右侧展示百分比数值
- 合计行：加粗，Fill-1 背景，不展示排名徽章和进度条
- 行高：约 44px（与首列一致）
- 横向滚动容器：`overflow-x: auto`，隐藏滚动条，页面不横向溢出

### 5.6 空状态 / 加载态 / 错误态

| 状态 | 展示位置 | 样式 |
|------|----------|------|
| 加载中 | 表格区域 | 骨架屏（Fill-2 灰色块，动画 shimmer） |
| 数据为空 | 表格区域 | 居中图标 + 「暂无数据」文案，Text-3 色 |
| 接口超时 | 表格区域 | 居中图标 + 「数据加载失败，请稍后重试」+ 重试按钮 |
| 无权限 | 页签 / 表格 | 页签置灰，表格展示「暂无权限」 |

## 8. 设计冻结规则

- demo 实现阶段不再向用户确认颜色、布局、组件选择或是否继续
- 缺失信息按同类页面（销量快报）、项目画像（营销驾驶舱）、Snow Mobile 规范推断
- **明确禁止的脑补**：
  - 不要把创新业务需求脑补成通用仪表盘（销售额、订单量、达成率、近7天趋势、TOP5区域排行）
  - 不要使用随机渐变、大面积彩色卡片、emoji 图标、营销式 Hero
  - 不要只做功能字段完整，却忽略项目真实结构模式
  - 不要用一堆卡片从上到下堆完所有信息，忽略 Banner 导航按钮切换
  - 不要把 Banner 导航按钮做成页面级 Tab（放在顶部导航栏下方）
  - 不要把子类分行列表做成横滑子维度切换（销量快报模式），本需求是"一个表格分行"
  - 不要添加趋势图、折线图、柱状图、饼图（用户明确不要图表）
  - 不要添加目标达成率、预算对比、创新业务占比等非本期指标
  - 演示页代码不要全是 PC px 思维，完全没有移动端 H5 和 uni-app 尺寸节奏
