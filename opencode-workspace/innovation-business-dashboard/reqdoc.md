# 创新业务看板 功能需求方案文档

> **版本**：v1.0
> **日期**：2026-06-26
> **适用系统**：华润雪花营销驾驶舱 H5
> **需求来源**：用户口述《创新业务看板需求》
> **优先级**：Banner 导航切换 / 整体汇总 / 子类分行列表 / 组织排名表格 / 组织层级页签 / 时间口径切换（均 P0）

> **页面视觉规格**：`./ui-spec.md`
> **demo链接**：`./demo.html`

## 1. 需求背景与目标

### 1.1 项目背景

华润雪花啤酒营销驾驶舱 H5 是面向内部销售、业务部、营销中心及管理层的移动端数据看板系统。现有看板已覆盖分档次、分品牌、分主力品种、其他品种等常规啤酒销量维度（销量快报模块），以及库存分析、经营分析等模块。

但**创新业务**（代工类、特色产品、桶啤）作为独立业务板块尚无专门看板。管理层目前查看创新业务销量需通过线下报表或口头汇报，存在以下问题：

- 创新业务包含代工类、特色产品、桶啤三大板块，各板块下还有细分品类（代加工 / 包销 / 定制 / 小品种 / 贸易产品 / 醴 / 高档桶啤 / 普通桶啤），线下报表难以一屏看清整体结构
- 缺少同期对比和同比增幅，无法快速判断各板块增长 / 下滑态势
- 无法按组织层级（总部 / 区域 / 营销中心 / 工厂）横向对比各组织的创新业务表现

### 1.2 业务目标

- 提供统一的创新业务看板入口，让管理层一屏看清创新业务大盘结构
- 支持按四大视角（整体 / 代工类 / 特色产品 / 桶啤）快速切换查看销量表现
- 支持组织层级切换（总部 / 区域 / 营销中心 / 工厂），实现横向排名对比
- 支持日 / 月 / 年三种时间口径切换，适配不同决策频率

### 1.3 用户目标

- **总部管理层**：每日 / 每月查看全国创新业务大盘，快速掌握代工类 / 特色产品 / 桶啤三大板块销量结构及同比变化
- **区域管理层**：查看所辖区域创新业务排名，横向对比各区域表现，定位增长 / 下滑区域
- **营销中心负责人**：查看所辖营销中心及工厂创新业务，了解下属各组织完成情况
- **业务部人员**：查看具体品类（代加工 / 包销 / 定制等）创新业务销量，精细化跟踪

### 1.4 本轮要解决的问题

1. 创新业务缺少专门看板，数据分散在线下报表中
2. 三大板块及细分品类的销量结构无法一屏看清
3. 缺少同期对比和同比增幅，无法快速判断增长 / 下滑态势
4. 无法按组织层级横向对比各区域 / 营销中心 / 工厂的创新业务表现
5. 日 / 月 / 年不同时间口径无法快速切换

### 1.5 不在本轮解决的问题

- 趋势图表（折线图 / 柱状图 / 饼图）
- 数据导出功能
- 数据下钻到具体客户 / 门店 / SKU
- 目标达成率 / 预算对比
- 实时数据刷新 / 推送
- 多组织层级同时对比
- 创新业务占比（相对总销量的份额）

## 3. 需求详情

### 3.1 Banner 导航按钮与视角切换

#### 3.1.1 适用角色与权限

- **全部角色**：均可查看和切换，数据范围按用户所属组织层级隔离

#### 3.1.2 触发入口

**入口 1 — 营销驾驶舱首页（本期主入口）**：
- 营销驾驶舱 H5 首页「创新业务看板」入口卡片，点击进入

**入口 2 — 直接链接（辅助入口）**：
- 通过分享链接或扫码直接进入创新业务看板页面

#### 3.1.3 页面与视图

> **页面视觉规格**：参考 `ui-spec.md`
> **界面原型**：参考 `demo.html` → 进入页面默认展示「整体」视角。以下为关键界面说明：

**创新业务看板 — 整体视角**：

![创新业务看板-整体视角](images/创新业务看板-整体视角.png)

- 页面标题：创新业务看板
- Banner 导航按钮：整体 | 代工类 | 特色产品 | 桶啤
- 时间口径：日 / 月 / 年
- 当前选中：整体

**创新业务看板 — 代工类视角**：

![创新业务看板-代工类视角](images/创新业务看板-代工类视角.png)

- Banner 导航按钮选中：代工类
- 子类分行列表：代加工 / 包销 / 定制

#### 3.1.4 交互步骤

1. 用户从营销驾驶舱首页点击「创新业务看板」进入
2. 默认选中「整体」视角，时间默认当前月，组织层级默认「区域」
3. 用户点击 Banner 内导航按钮切换视角（整体 / 代工类 / 特色产品 / 桶啤）
4. 切换后，分类汇总表格、组织排名表格同步更新
5. 用户可继续切换组织层级页签和时间口径

#### 3.1.5 关键状态

| 状态 | 说明 / 展示方式 |
|------|----------------|
| 整体选中 | 导航按钮「整体」高亮（Brand-6 底色，白色文字），其余按钮默认态 |
| 代工类选中 | 导航按钮「代工类」高亮，其余按钮默认态 |
| 特色产品选中 | 导航按钮「特色产品」高亮，其余按钮默认态 |
| 桶啤选中 | 导航按钮「桶啤」高亮，其余按钮默认态 |
| 加载中 | 表格区域展示骨架屏或 Loading |
| 数据为空 | 表格区域展示空状态「暂无数据」 |

#### 3.1.6 业务规则

- 视角切换时保留当前组织层级页签和时间口径状态
- 视角切换后，分类汇总表格行数和内容同步变化
- 默认进入「整体」视角，时间默认当前月，组织层级默认「区域」
- 导航按钮为单选互斥，同一时间只能选中一个视角

#### 3.1.7 数据输入输出

- **输入**：视角标识（overall / oem / specialty / draught）、时间口径（day / month / year）、组织层级（hq / region / mc / factory）、当前选中日期
- **输出**：分类汇总表格数据 + 组织排名表格数据

#### 3.1.8 异常或边界情况

- **接口超时**：10s 后展示兜底提示「数据加载失败，请稍后重试」，保留当前筛选状态
- **数据为空**：表格区域展示空状态，不展示空表格骨架
- **同比增幅为无穷大（同期为 0）**：展示「—」或「NEW」标识，不作除法计算

#### 3.1.9 开发必看

> Banner 导航按钮是本需求的核心交互，区别于销量快报的页面级 Tab。导航按钮放在 Banner 区域内部，不是页面级 Tab。切换时需保留组织层级页签和时间口径状态，避免用户重复操作。接口请求需携带视角标识参数，后端按视角返回不同层级的数据聚合结果。

#### 3.1.10 重点注意

> 切换视角后，分类汇总表格的行数和行内容都会变化（整体=3行、代工类=3行、特色产品=3行、桶啤=2行），表格高度会动态变化，需做好布局适配。组织排名表格的数据范围也随视角变化（整体=创新业务总量、代工类=仅代工类、特色产品=仅特色产品、桶啤=仅桶啤）。

#### 3.1.11 风险提醒

> Banner 导航按钮与销量快报的页面级 Tab 在视觉上需有区分，避免用户混淆。建议导航按钮采用胶囊按钮组样式，放在 Banner 背景图上方的内容流中。

### 3.2 整体视角 — 三大分类汇总对比

#### 3.2.1 适用角色与权限

- **全部角色**：均可查看，数据范围按用户所属组织层级隔离

#### 3.2.2 触发入口

**入口 1 — 默认进入（本期主入口）**：
- 进入创新业务看板默认展示「整体」视角

**入口 2 — 导航按钮切换（辅助入口）**：
- 从其他视角点击「整体」导航按钮切换回来

#### 3.2.3 页面与视图

> **页面视觉规格**：参考 `ui-spec.md`
> **界面原型**：参考 `demo.html` → 点击「整体」导航按钮查看

**整体视角 — 三大分类汇总表格**：

![创新业务看板-整体视角](images/创新业务看板-整体视角.png)

- 表格标题：创新业务分类汇总
- 表格字段：

| 字段 | 说明 |
|------|------|
| 分类名称 | 代工类 / 特色产品 / 桶啤 |
| 当期（kL） | 当前时间口径下的销量，单位千升，保留1位小数 |
| 同期（kL） | 去年同期销量，单位千升，保留1位小数 |
| 同比增减（kL） | 当期 - 同期，正数绿色、负数红色 |
| 同比增幅 | 同比增减 / 同期 × 100%，正数绿色带↑、负数红色带↓ |

#### 3.2.4 交互步骤

1. 用户进入或切换到「整体」视角
2. 页面展示三大分类汇总表格，默认数据为当前月、当前组织层级
3. 用户可切换时间口径（日 / 月 / 年）查看不同时间维度的汇总
4. 用户可切换组织层级页签查看不同层级的组织排名（见 3.7）

#### 3.2.5 关键状态

| 状态 | 说明 / 展示方式 |
|------|----------------|
| 数据加载中 | 表格展示骨架屏 |
| 数据正常 | 表格展示3行数据（代工类 / 特色产品 / 桶啤），同比正负着色 |
| 同期为0 | 同比增幅展示「NEW」标识，同比增减展示当期值 |
| 数据为空 | 表格区域展示「暂无数据」 |

#### 3.2.6 业务规则

- 整体汇总 = 代工类 + 特色产品 + 桶啤 的合计
- 三大分类数据为各分类下所有子类的销量合计
- 当期 / 同期 / 同比增减 / 同比增幅 四项指标必须同时展示
- 同比增减 = 当期 - 同期，保留1位小数
- 同比增幅 = 同比增减 / 同期 × 100%，保留1位小数，带%符号
- 同比增幅正数带↑符号绿色，负数带↓符号红色

#### 3.2.7 数据输入输出

- **输入**：时间口径（day / month / year）、当前日期、组织层级、用户组织权限范围
- **输出**：3行汇总数据（代工类 / 特色产品 / 桶啤 × 当期 / 同期 / 同比增减 / 同比增幅）

**示例数据（月口径，当前组织层级：区域，2026年6月）**：

| 分类 | 当期（kL） | 同期（kL） | 同比增减（kL） | 同比增幅 |
|------|-----------|-----------|---------------|----------|
| 代工类 | 62,580.0 | 55,230.0 | +7,350.0 | ↑ 13.3% |
| 特色产品 | 48,920.0 | 41,680.0 | +7,240.0 | ↑ 17.4% |
| 桶啤 | 38,500.0 | 33,290.0 | +5,210.0 | ↑ 15.7% |
| **合计** | **150,000.0** | **130,200.0** | **+19,800.0** | **↑ 15.2%** |

#### 3.2.8 异常或边界情况

- **同期为0**：同比增幅展示「NEW」，同比增减展示当期值（绿色）
- **当期为0**：当期展示0.0，同比增减展示-同期值（红色），同比增幅展示-100.0%
- **当期和同期均为0**：全部展示0.0或「—」
- **数据未出**：展示「数据待更新」提示

#### 3.2.9 开发必看

> 整体视角的汇总表格必须包含合计行（第4行），合计行加粗展示。合计行不参与组织排名，仅作汇总参考。后端接口需返回三大分类的合计数据，前端不可自行相加（避免浮点精度问题）。

#### 3.2.10 重点注意

> 同比增幅的着色规则：正数（增长）用 Success-4（#2EB82E）绿色 + ↑ 符号；负数（下滑）用 Danger-4（#F53D3D）红色 + ↓ 符号。零增长展示灰色「0.0%」。这与销量快报的着色规则保持一致。

#### 3.2.11 风险提醒

> 三大分类的销量合计不一定等于营销驾驶舱总销量（创新业务是总销量的一部分）。需在表格标题或说明文案中明确"创新业务"范围，避免用户误认为是全品销量。

- 第4名及以后使用普通数字序号
- 固定首列：组织名称列固定，右侧数据列横向滚动
- 增量贡献进度条：宽度 = 当期 / 当期最大组织当期 × 100%

- 表格数据范围随视角变化：整体=创新业务总量、代工类=仅代工类、特色产品=仅特色产品、桶啤=仅桶啤

#### 3.6.8 异常或边界情况

- **组织数量为1（工厂层级）**：展示单行数据 + 上级汇总行
- **当期为0的组织**：排名靠后，当期展示0.0，进度条为空
- **同比增幅无穷大**：展示「NEW」
- **组织名称过长**：独占一行展示，数据行用空列占位对齐表头（参考项目本地经验 dealer-name-separate-row）

#### 3.6.9 开发必看

> 组织排名表格是本需求信息密度最高的模块。固定首列 + 横向滚动 + 进度条 + Top3 徽章 + 正负着色，需同时处理。后端接口需返回排名、组织名、当期、同期、同比增减、同比增幅六项数据，增量贡献进度条由前端计算（当期 / 当期最大值）。表格必须放入横向滚动容器，页面本身不能横向溢出。

#### 3.6.10 重点注意

> 固定首列与右侧滚动区的行高必须一致，否则表格错位。组织名称过长时独占一行（参考项目本地经验 dealer-name-separate-row）。排名徽章 Top3 使用 Brand-6 蓝色圆形背景 + 白色数字，第4名及以后使用 Text-4 灰色数字。

#### 3.6.11 风险提醒

> 增量贡献进度条的计算口径（按当期 / 当期最大值 vs 按当期 / 合计）需与后端确认。暂按"当期 / 当期最大组织当期 × 100%"设计，使最大组织进度条满格。如改为"当期 / 合计 × 100%"，所有进度条都不会满格，视觉效果不同。

| 进度条计算逻辑 | 前端计算：当期 / 当期最大值 × 100% |
| 空状态 / 加载态 | 骨架屏、空状态、接口超时兜底 |
| 生僻字"醴"字体适配 | 指定 PingFang SC，测试生僻字渲染 |

### 4.2 后端开发事项

| 事项 | 说明 |
|------|------|
| 创新业务销量聚合接口 | 按视角（整体 / 代工类 / 特色产品 / 桶啤）× 时间口径（日 / 月 / 年）× 组织层级返回汇总数据 |
| 组织排名接口 | 按视角 × 时间口径 × 组织层级返回组织排名列表 |
| 组织层级权限鉴权 | 按用户角色校验可查看的组织层级和数据范围 |
| 同比计算逻辑 | 当期 / 同期 / 同比增减 / 同比增幅，处理同期为0的特殊情况 |
| 合计行计算 | 后端返回合计数据，前端不可自行相加（避免浮点精度） |

### 4.3 接口或数据依赖

| 接口 / 数据 | 说明 |
|-------------|------|
| 创新业务销量汇总接口 | 入参：视角标识、时间口径、日期、组织层级、用户Token；出参：分类汇总行列表 + 合计 |
| 组织排名接口 | 入参：视角标识、时间口径、日期、组织层级、用户Token；出参：组织排名列表 |
| 组织主数据 | 总部 / 区域 / 营销中心 / 工厂 四级组织树及名称 |
| 用户权限数据 | 用户所属组织层级、可查看的数据范围 |
| 现有营销驾驶舱框架 | uni-app + Vue 3，复用现有 HTTP 请求封装、组织鉴权、页面框架 |

### 4.4 状态管理或流程编排

- 页面状态：当前视角、当前时间口径、当前日期、当前组织层级
- 视角切换 → 同步刷新分类汇总表格 + 组织排名表格
- 组织层级切换 → 同步刷新组织排名表格（保留视角和时间）
- 时间口径切换 → 同步刷新所有表格（保留视角和组织层级）
- 接口请求策略：视角 + 时间 + 组织层级任一变化时重新请求，需做防抖处理避免频繁请求

### 4.5 补充事项

- **埋点**：页面访问埋点、视角切换埋点（记录从哪个视角切到哪个视角）、组织层级切换埋点、时间口径切换埋点
- **日志**：接口请求 / 异常日志
- **权限**：沿用营销驾驶舱 H5 现有组织权限体系，不新增角色

## 6. 风险与处理策略

| 风险 | 影响 | 处理策略 |
|------|------|----------|
| **需求理解风险**：Banner 导航按钮与销量快报页面级 Tab 混淆 | 页面结构实现偏差 | 明确导航按钮放在 Banner 内部，采用胶囊按钮组样式，与页面级 Tab 视觉区分；ui-spec.md 中固化此设计 |
| **交互或流程风险**：4个组织层级页签在移动端宽度不足 | 页签挤压、文字截断 | 暂按分段器设计；如宽度不足，备选方案：缩小字号 / 减少内边距 / 改下拉选择器；列待确认 |
| **技术依赖风险**：创新业务数据是否已在数据中台落库 | 接口设计、数据来源阻塞 | 需确认数据来源系统；暂按营销驾驶舱统一数据中台设计，如数据未落库需新增数据采集任务 |
| **技术依赖风险**：4级组织聚合（总部 / 区域 / 营销中心 / 工厂）是否已支持 | 后端聚合逻辑、排期 | 需确认数据中台是否支持4级聚合；如不支持需新增聚合逻辑 |
| **数据一致性风险**：创新业务合计 ≠ 营销驾驶舱总销量 | 用户误解数据 | 表格标题明确"创新业务"范围，不加"总销量"字样；reqdoc 中已注明 |
| **数据一致性风险**：同期为0时同比增幅计算 | 展示异常、除零错误 | 前端展示「NEW」标识，后端返回 null 或特殊标记，前端不作除法计算 |
| **排期风险**：组织层级从2级扩展为4级，后端开发量增加 | 排期延期 | 优先实现「区域」层级（P0），「总部 / 营销中心 / 工厂」可分期交付 |
| **兼容性风险**：生僻字"醴"在部分机型渲染异常 | 文字显示异常 | 指定 PingFang SC 字体，测试 iOS 12 / Android 8 最低版本；如异常不可替换为拼音 |

## 7. 非功能需求

| 维度 | 要求 |
|------|------|
| **性能** | 页面加载 < 2s；数据查询接口 < 1s（10s 超时兜底）；表格渲染 < 500ms；视角切换到数据展示 < 800ms |
| **安全** | 沿用营销驾驶舱 H5 现有鉴权体系；后端按用户组织层级校验数据范围；前端页签控制为辅助手段，不可作为唯一权限屏障 |
| **稳定性** | 接口超时 10s 后展示「数据加载失败，请稍后重试」兜底提示，保留当前筛选状态；数据为空展示空状态 |
| **易用性** | 同比正负色区分明显（绿色 / 红色）；Top3 徽章视觉突出；进度条直观展示占比；表格支持横向滚动 |
| **可维护性** | 子类名称（代加工 / 包销 / 定制 / 小品种 / 贸易产品 / 醴 / 高档桶啤 / 普通桶啤）前端硬编码，不从接口动态获取名称；分类层级结构固定 |
| **兼容性** | 适配 iOS / Android 主流机型，最低支持 iOS 12 / Android 8；以 375px 移动端为主要设计基准，同时检查 390px 和 430px |
| **可观测性** | 页面访问埋点、视角切换埋点（from → to）、组织层级切换埋点、时间口径切换埋点；接口请求 / 异常日志 |
