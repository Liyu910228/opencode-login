对于需要给用户查看结果的 HTML / 前端 demo / 静态页面，遵守以下规则：

1. 不要让用户打开本机路径，例如 `/tmp/...`、`file://...`、`open ...`、`xdg-open ...`。
2. 不要只返回 `localhost`、`127.0.0.1` 或仅本机可访问的端口地址。
3. 需要给用户可直接访问的公网预览链接。
4. 默认把预览产物发布到**中央预览目录** `/srv/opencode-previews/<slug>/`（opencode 用户拥有，任何项目都可写，不会出现“创建目录失败”）。
5. `<slug>` 必须以 `pub-` 开头，后接描述性英文短标识，例如 `pub-sales-flash-demo`；不与已有项目目录名冲突，预览路由才能正确命中中央目录。
6. 单页面 demo 使用 `/srv/opencode-previews/<slug>/index.html` 作为入口；多文件静态页面把全部依赖文件都放进同一个 `/srv/opencode-previews/<slug>/` 目录内，并使用相对路径引用资源。
7. 公网链接格式为：`http://119.45.222.120:4096/preview/<slug>/`。
8. 如果当前实现只能在开发服务器里运行，需要额外生成一个可静态访问的预览版本，再把公网链接给用户。
9. 仅当明确需要按项目隔离预览时，才改用当前项目目录下的 `./.public-preview/<slug>/`，并返回 `http://119.45.222.120:4096/preview/<当前项目目录名>/<slug>/`；此时当前项目目录必须对 opencode 用户可写，否则一律用中央目录。

默认优先交付“可以直接点开查看”的版本，而不是只给本机运行说明。
