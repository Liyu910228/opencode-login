# 组件规则

## 按钮

按钮用于明确命令。每个页面区块或底部操作区只保留一个主要操作。

### 变体结构

| 属性 | 可选值 |
| --- | --- |
| 形状 | 圆角 |
| 样式 | 主要按钮、辅助按钮、描边按钮 |
| 尺寸 | 特大、大（常规）、中、小 |
| 状态 | 默认、点击、禁用 |
| 类型 | 默认、危险、警告 |

### 尺寸与排版

| 尺寸 | 组件尺寸 | 内边距 | 圆角 | 字体 |
| --- | --- | --- | --- | --- |
| 特大 | 112 x 46 | 12px 24px | 22px | 16/Medium |
| 大（常规） | 96 x 38 | 8px 16px | 20px | 16/Medium |
| 中 | 80 x 30 | 5px 12px | 20px | 14/Medium |
| 小 | 64 x 24 | 3px 8px | 20px | 12/Regular |

按钮内容水平垂直居中，文本单行居中，内容间距 `gap: 4px`。

### 样式与颜色

| 样式/类型 | 默认态 |
| --- | --- |
| 主要按钮/默认 | 背景 `主要颜色/primary`，文字 `文字与图标-text/纯白` |
| 主要按钮/危险 | 背景 `功能色/危险-danger/danger`，文字 `文字与图标-text/纯白` |
| 主要按钮/警告 | 背景 `功能色/警告-warning/warning`，文字 `文字与图标-text/纯白` |
| 辅助按钮/默认 | 背景 `主要颜色/light-default`，文字 `主要颜色/primary` |
| 描边按钮/默认 | 1px 内描边 `描边色-border/focus-border`，文字 `主要颜色/primary` |

状态色：

- 默认主按钮使用 `主要颜色/primary`。
- 点击态默认主按钮使用 `主要颜色/active`。
- 禁用态默认主按钮使用 `主要颜色/disabled`。
- 危险按钮使用 `功能色/危险-danger/danger`。
- 警告按钮使用 `功能色/警告-warning/warning`。

规则：

- 主按钮只用于当前区域最重要的业务动作。
- 次按钮/默认按钮用于辅助动作。
- 文字按钮或幽灵按钮用于低强调动作。
- 操作不可用时展示禁用态；必要时在附近说明不可用原因。
- 按钮文案要简短、动词化，例如“提交”“确认收货”“重新加载”“筛选”。
- 同一视觉分组内避免出现多个主按钮。
- 选择尺寸时，页面底部主操作优先用大（常规）或特大；列表行内操作优先用中或小。
- 描边按钮用于可见但非最高优先级的操作；辅助按钮用于弱化但仍需品牌感的操作。
- 危险/警告类型只用于真实风险或提醒操作，不用于普通强调。

移动端底部操作：

- 高频提交、审批、确认流程优先使用底部固定操作栏。
- 危险操作要和正向主操作拉开视觉距离。

## 徽标

徽标用于紧凑展示状态、数量和标签。

规则：

- 徽标表达业务状态，不做纯装饰。
- 徽标文字要短，只放状态、数量或紧凑分类。
- 状态色要匹配业务含义：成功、警告、错误、处理中、中性。
- 单个列表行不要堆叠过多徽标。
- 关键流程状态要靠近实体标题或关键字段。

常见状态示例：

- 待处理
- 审批中
- 已完成
- 已驳回
- 异常

## 分页器

分页器或加载更多只在数据源分页、列表可能较长时使用。

规则：

- 移动端根据产品模式选择简单分页、“加载更多”或无限滚动。
- 当页码、总数或加载状态会影响用户判断时，要保持可见。
- 短固定列表不要使用分页器。
- 切换分页时保留筛选和选项卡状态。

## 表格

表格用于移动端看板、排行、指标明细和多字段对比。只有当数据需要按行列精确比较时使用表格；普通对象集合优先使用列表。

### 组件能力

表格可按业务需要组合以下能力：

| 能力 | 使用场景 | 是否必需 |
| --- | --- | --- |
| 基础表格 | 字段少、无需横向滚动的明细 | 基础能力 |
| 固定首列 | 行身份需要始终可见，如区域、门店、人员、商品 | 按需 |
| 横向滚动 | 列数超过屏宽或需要多指标对比 | 按需 |
| 分组表头 | 字段存在父子层级，如大类 + 子指标 | 按需 |
| 排序 | 列支持升降序比较 | 按需 |
| 帮助说明 | 指标口径需要解释 | 按需 |
| 状态标识 | 行有风险、等级、异常、达成状态 | 按需 |
| 下钻入口 | 行可进入详情 | 按需 |

生成表格前先判断需要哪些能力，不要把所有能力机械堆叠到每张表里。

### 基础结构

- 表格由表头、数据行和可选辅助能力组成。
- 表头用于解释列含义，数据行用于承载真实业务数据。
- 移动端表格优先保持扫读稳定，不为了塞满一屏强行压缩字段。
- 当列数超过屏宽时，保留横向滚动。
- 当行身份重要且列数较多时，固定首列。
- 表格必须放在横向滚动容器中；表格总宽可以大于屏幕，但页面本身不能横向溢出。
- 表格放在卡片内时，外层卡片负责标题、说明和操作；表格内部只承载字段和数据。

### 防溢出实现

- 外层容器负责横向滚动，页面根容器不得被表格撑出横向滚动。
- 每列设置可读的最小宽度，不要为了塞进一屏把列压缩到不可读。
- 数字、日期、百分比、单位组合默认不换行。
- 长文本默认单行省略；业务必须完整展示时才允许换行，并保持整行高度一致。
- 表头和数据列必须一一对齐，不能因为内容过长造成列错位。
- 固定首列存在时，首列与右侧滚动区的数据行高度必须一致。
- 表格内容不得溢出单元格边界；出现溢出时优先调整列宽、截断策略或横向滚动。

### 基础视觉

表格、表头和分组表头的基础视觉必须固定，不随业务页面自由发挥。

| 对象 | 规范 |
| --- | --- |
| 表格背景 | `填充-fill/纯白` |
| 表头背景 | `主要颜色/light-default` |
| 表头文字 | `10/Regular` 或 `12/Regular`，颜色 `文字与图标-text/1` |
| 数据文本 | `14/Regular`，颜色 `文字与图标-text/0` |
| 数字文本 | 14px 数字字体或等宽数字，颜色 `文字与图标-text/0` |
| 次要信息 | `12/Regular`，颜色使用次要文字色 |
| 分隔线/单元格边框 | 1px，使用弱分隔线色，优先 `描边色-border/border-透明`；无该 token 时使用 `Divider Color` |
| 排序/帮助/下钻图标 | 12 x 12，颜色使用次要文字色；激活排序方向可使用 `主要颜色/primary` |
| 状态点 | 8 x 8，Radius-circle，颜色遵守状态语义 |

规则：

- 表头、分组表头和固定首列表头使用同一表头背景和边框规则。
- 表格内部不使用卡片阴影、渐变、装饰背景或额外描边风格。
- 表格分隔线只用于帮助行列对齐，不要使用过重边框。
- 同一张表内字体、边框、表头背景和图标颜色必须一致。
- 业务差异只能影响字段、列宽、能力组合和状态语义，不能改变表格基础视觉。

### 尺寸建议

| 项目 | 建议 |
| --- | --- |
| 单层表头 | 32-40px |
| 分组一级表头 | 28-34px |
| 分组二级表头 | 36-48px |
| 数据行 | 40-48px |
| 固定首列 | 64-96px |
| 紧凑数值列 | 44-64px |
| 常规数值列 | 56-80px |
| 文本/日期列 | 80-120px |

以上为移动端可读范围，不是固定参数。根据字段长度、数据密度和屏幕宽度微调，但不能牺牲可读性和列对齐。

### 表头

- 表头背景固定使用 `主要颜色/light-default`。
- 表头文字使用 `10/Regular` 或 `12/Regular`，颜色固定使用 `文字与图标-text/1`。
- 表头文案要短，优先使用业务字段名。
- 表头高度根据层级决定：单层表头使用紧凑高度，分组表头按层级叠加高度。
- 不要为了放长文案把表头撑得过高；长字段应缩短命名或使用帮助说明。
- 有排序能力时才显示排序图标；当前排序态需要明确。
- 有指标解释时才显示帮助图标，不要每列都机械添加。

### 分组表头

当字段存在父子关系时使用分组表头，例如“大类 + 子指标”。

规则：

- 分组表头只用于真实字段层级，不用于普通并列字段。
- 一级表头承载字段大类。
- 二级表头承载具体指标。
- 一级表头必须横跨其下所有子列。
- 子列表头必须与下方数据列一一对齐。
- 固定首列存在时，首列表头应纵向合并，和分组表头总高度一致。
- 分组表头的层级不宜超过 2 层；超过 2 层时优先调整信息架构或拆分表格。
- 分组表头可以横向滚动，但不能导致首列与数据行错位。
- 分组表头出现后，不要把同一层级字段拆成多个碎片卡片。

### 数据行

- 数字列右对齐，便于上下比较。
- 文本列左对齐，便于扫读名称。
- 状态列居中或紧凑左对齐。
- 同一列的对齐方式必须一致。
- 数据行高度保持稳定。
- 行内不堆叠复杂说明；复杂内容进入详情。
- 空值使用短横线或业务约定占位，不使用无意义文本。

### 固定首列

首列用于承载行身份，例如事业部、营销中心、城市、门店、人员或商品。

规则：

- 固定首列只在行身份需要始终可见时使用。
- 固定首列与右侧滚动区的数据行高度必须一致。
- 固定首列可包含排名、状态点、名称和下钻箭头，但只放真实业务需要的信息。
- 排名只用于真实排行表；普通明细表不要强行添加排名。
- 状态点只用于真实风险、等级、异常或达成状态区分。
- 行可进入详情时，可在首列或行尾放右箭头；不可点击行不要放箭头。

### 辅助能力

排序：

- 只有可排序字段显示排序图标。
- 当前排序态需要用颜色或方向清楚表达。
- 不可排序字段不要显示排序图标。

帮助说明：

- 只有指标口径需要解释时显示帮助图标。
- 帮助图标靠近字段名，不打断表头对齐。
- 不要每列表头都加帮助图标。

状态标识：

- 状态点、颜色、徽标只用于真实风险、等级、异常或达成状态。
- 状态色必须符合语义，不只按正负号机械上色。
- 同一列的正负方向含义必须一致。

下钻：

- 行可进入详情时，可在首列或行尾放右箭头。
- 不可点击行不要放箭头。
- 下钻入口不能影响文本和数字列对齐。

### 使用规则

- 表格适合看板下钻、排行榜、多指标对比，不适合承载复杂表单。
- 列数超过屏宽时保留横向滚动，不要改成多个碎片卡片。
- 看板表格和多指标对比表必须检查横向溢出；不得让表格撑破页面宽度。
- 表格需要分页时优先使用加载更多或简单分页；短排行不使用分页。
- 字段过多时优先分组、筛选或下钻，不要无限增加窄列。

## 选项卡

选项卡用于同一页面上下文中的同级分类或状态切换。

### 变体结构

| 属性 | 可选值 |
| --- | --- |
| 类型 | 基础用法、标签卡、分段器、卡片、竖排卡片 |
| 数量 | 2、3、4、5 |

### 基础用法

用于页面顶部或内容区内的普通横向切换。

| 项目 | 规格 |
| --- | --- |
| 容器 | 375 x 44，横向排列 |
| 单项 | 64 x 42 |
| 内边距 | 10px 16px |
| 选中项文字 | `16/Medium`，`主要颜色/primary` |
| 未选中项文字 | `16/Regular`，`文字与图标-text/0` |
| 选中指示线 | 24 x 2，圆角 1px，`主要颜色/primary` |

### 标签卡

用于较强的筛选或状态切换，适合短标签。

| 项目 | 规格 |
| --- | --- |
| 容器 | 375 x 34，横向排列，间距 16px |
| 单项 | 96 x 34 |
| 内边距 | 6px 16px |
| 圆角 | 80px |
| 选中项 | 背景 `主要颜色/primary`，文字 `文字与图标-text/纯白`，字体 `16/Medium` |
| 未选中项 | 背景 `填充-fill/0-透明`，文字 `文字与图标-text/0`，字体 `16/Regular` |

### Banner 胶囊标签卡

用于看板正文内容层中的指标口径、时间口径或对比方式切换。

| 项目 | 规格 |
| --- | --- |
| 容器 | 只位于看板正文内容层，横向排列，宽度等于正文内容区可用宽度 |
| 选项间距 | 固定 8px |
| 单项宽度 | `(容器宽度 - 8px * (选项数量 - 1)) / 选项数量` |
| 单项高度 | 32px |
| 内边距 | 5px 12px |
| 圆角 | Radius-pill |
| 选中项 | 背景 `填充-fill/纯白`，文字 `主要颜色/primary`，字体 `14/Medium` |
| 未选中项 | 背景白色低透明，文字 `文字与图标-text/纯白`，字体 `14/Regular`，可使用 1px 白色低透明描边，并启用背景模糊效果 |

规则：

- 只用于看板维度切换，不用于首个内容卡片内部或普通筛选区。
- 选项宽度不按文案自适应，必须按容器可用宽度、固定间距和选项数量等分计算。
- 容器宽度由正文内容层左右边距决定；不要让选项超出正文内容区。
- 2-4 个选项优先使用等分胶囊；超过 4 个时优先精简维度或改为横向滚动，不要压缩到不可读。
- 每个选项必须是独立胶囊，选项之间必须保留 8px 间距；不要把多个选项合并到一整条半透明底板里。
- 选中态使用白色胶囊，不使用主色实底。
- 未选中态保持弱化，但在 页面顶部 Banner 背景上必须可读；不要因为透明度过低导致未选中项文字消失。
- 可使用背景模糊提升未选中项文字识别度，但不要叠加整条实色遮罩。
- 不要把 Banner 胶囊标签卡移到 正文内容层之外。

### 分段器

用于强互斥、短选项的切换，通常为等分布局。

| 项目 | 规格 |
| --- | --- |
| 容器 | 375 x 34 |
| 圆角 | 4px |
| 描边 | 1px，`描边色-border/focus-border` |
| 排列 | 横向等分，相邻项边线重叠 |
| 内边距 | 6px 16px |
| 选中项 | 背景 `主要颜色/primary`，文字 `文字与图标-text/纯白`，字体 `16/Medium` |
| 未选中项 | 背景 `填充-fill/0-透明` 或 `填充-fill/纯白`，文字 `主要颜色/primary`，字体 `16/Regular` |

### 卡片

用于带内容承载区的选项切换。

| 项目 | 规格 |
| --- | --- |
| 容器 | 375 x 104，纵向结构 |
| 标签区 | 高 40 |
| 内容区 | 高 64 |
| 卡片圆角 | 8px |
| 选中项文字 | `14/Medium`，`主要颜色/primary` |
| 未选中项文字 | `14/Regular`，`文字与图标-text/0` |
| 选中指示线 | 24 x 2，圆角 1px，`主要颜色/primary` |
| 内容区背景 | `填充-fill/纯白` |

### 竖排卡片

用于左侧分类、右侧内容的纵向切换结构。

| 项目 | 规格 |
| --- | --- |
| 容器 | 375 x 600 |
| 左侧选项区 | 宽 120，背景 `填充-fill/0-透明` |
| 右侧内容区 | 宽 255，背景 `填充-fill/纯白` |
| 选中项 | 120 x 56，内边距 16px，背景 `填充-fill/纯白` |
| 未选中项 | 120 x 52，内边距 16px |
| 选中项文字 | `14/Medium`，`主要颜色/primary` |
| 未选中项文字 | `14/Regular`，`文字与图标-text/0` |
| 选中指示线 | 2 x 20，圆角 1px，`主要颜色/primary` |

规则：

- 选项卡用于不离开当前页面的内容切换。
- 标签文案要短，分类之间要互斥。
- 生成页面前先确定每组选项卡的层级和用途：全局切换、模块内切换、状态切换或筛选切换。
- 看板页页面级顶部 Tab 固定使用 `基础用法`，位置在顶部导航栏下方、Banner 上方。
- 看板页 看板维度 Tab 固定使用 `Banner 胶囊标签卡`，位置在正文内容层。
- 看板正文模块内如需切换，只能从基础用法、标签卡、分段器、卡片、竖排卡片中选择合适变体，不要创造组件规则中不存在的新样式。
- 同一页面中，同一层级、同一交互目的的选项卡只能使用一种变体。
- 全局切换、局部筛选、状态切换可以使用不同变体，但必须有明确层级差异。
- 不要因为不同区块单独设计，导致同级选项卡出现两套视觉语言。
- 当状态分组是核心工作流时，可使用“全部、待处理、进行中、已完成”等选项卡。
- 不相关的目标页面不要用选项卡承载，应使用导航。
- 选中态必须清晰。
- 如果业务需要对比，切换选项卡时应保留滚动或筛选状态。
- 2-5 个同级选项可直接使用组件变体；超过 5 个时优先考虑横向滚动、筛选入口或重新分组。

## 组件选择

当需求可以用已有组件表达时，优先使用已有组件。只有满足以下情况时，才建议新增组件或变体：

- 现有变体无法表达该业务状态；
- 该交互会在多个页面重复出现；
