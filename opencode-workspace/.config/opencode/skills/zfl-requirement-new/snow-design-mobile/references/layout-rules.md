# 布局规则

本文件用于控制页面布局、间距、层级和响应式稳定性。它不定义组件样式，也不替代具体页面模板；页面类型规则仍以对应 pattern 为准。

## 布局目标

- 先建立业务层级，再摆放组件。
- 页面应紧凑、清晰、可扫读，不做营销式大留白。
- 同一页面内只使用少量稳定间距，避免随机留白。
- 所有内容必须在 375/390/430px 移动端宽度下不重叠、不溢出、不漂移。

## 页面框架

- 移动端页面优先使用单列流式布局。
- 页面左右安全边距默认使用 `Space-4`。
- 顶部导航、页面内容、底部操作、底部导航各司其职，不互相承载对方职责。
- 页面主体从上到下按“当前上下文、关键结论、主体内容、辅助内容、操作反馈”组织。
- 页面级通栏区域可以突破内容边距；普通内容区应保持统一左右边距。

## 间距节奏

- 页面级区块之间优先使用 `Space-5` 或 `Space-6`。
- 普通模块之间优先使用 `Space-4` 或 `Space-5`。
- 模块标题与内容之间优先使用 `Space-2` 或 `Space-3`。
- 卡片内边距默认使用 `Space-4`。
- 列表行内部间距优先使用 `Space-2` 到 `Space-4`。
- 图标与文字、徽标与文字之间优先使用 `Space-1`。
- 同一页面优先控制在 3-5 个间距 token 内。
- 不要按屏幕宽度等比放大垂直间距；移动端 375/390/430px 下同类间距应保持一致。

## 层级规则

- 一级页面标题或模块标题必须清楚，不用装饰性大字替代业务标题。
- 模块层级最多保持 3 层：页面级、模块级、行/字段级。
- 同级模块使用一致的标题区、内边距和操作入口位置。
- 主要信息靠前、靠左或靠上；次要信息使用较弱文字层级。
- 操作入口靠近对应内容，不漂浮到无关区域。

## 对齐规则

- 页面标题、模块标题、内容左边界保持对齐。
- 同一模块内图标、标题、数值、状态和操作入口按稳定轴线对齐。
- 数字列右对齐，便于比较。
- 文本列左对齐，便于扫读。
- 状态徽标靠近状态对象，不脱离上下文。
- 同类卡片的高度、标题位置、内边距和操作区位置尽量一致。

## 容器规则

- 页面背景、卡片、列表、表格、表单字段要有明确层级，不靠阴影堆叠。
- 普通内容卡片使用 `Radius-8`；控件类容器按组件规则使用对应圆角。
- 卡片之间不要过度留白，也不要贴得像同一个容器。
- 通栏内容、Banner、底部导航等页面级区域不受普通内容左右边距限制。

## 响应式规则

- 以 375px 移动端为主要设计基准，同时检查 390px 和 430px。
- 横向空间增加时，优先增加内容可读宽度，不要等比放大字号、圆角和垂直间距。
- 固定格式元素必须有稳定尺寸或约束，例如 Tab、图标容器、表格列、按钮、徽标。
- 多列指标在窄屏保持可读；文字过长时优先换行、截断或调整信息层级，不压缩到不可读。
- 表格、多指标对比、宽字段明细通过横向滚动解决，不让页面根容器横向溢出。

## 生成流程

生成页面时先完成布局判断：

1. 判断页面类型和主要任务。
2. 确定首屏最重要的信息和操作。
3. 划分页面级区块和模块顺序。
4. 为每个模块选择承载方式：列表、卡片、表格、表单、图表或状态块。
5. 确定模块间距、模块内边距和对齐轴线。
6. 再套用基础规范、页面模板和组件规则。

## 不合格情况

- 页面间距像随机生成，缺少稳定节奏。
- 关键内容被装饰、空白或大标题挤出首屏。
- 同级模块内边距、标题位置或操作入口不一致。
- 文字、按钮、图标、表格或卡片发生重叠和溢出。
- 宽屏预览时垂直间距被等比放大，页面显得松散。
- 组件样式正确但模块层级、对齐和信息顺序混乱。
