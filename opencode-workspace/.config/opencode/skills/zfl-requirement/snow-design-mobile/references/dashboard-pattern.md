# 看板页硬模板

生成看板、数据看板、月报、日报、经营分析、库存分析、销量分析、风险监控页面时，必须先套用本模板，再补充业务内容。

## 生成顺序

1. 判断页面是否属于看板场景；命中后不要从普通列表页或详情页结构开始。
2. 读取 `dashboard-banners.md`，从 COS 目录素材池随机选择 1 张 Banner 原图。
3. 按需放置页面级顶部 Tab；它位于顶部导航栏下方、正文内容上方。
4. 使用页面顶部背景 Banner；Banner 只作为页面背景视觉层，不作为布局容器。
5. 看板名称、统计时间、可选看板维度 Tab、首个内容卡片、第二个内容卡片和后续模块全部属于正文内容层，按正常文档流从上到下排列。
6. 首个内容卡片放核心总览；后续按 KPI、风险提醒、趋势、排行、表格明细、下钻入口排列。
7. 交付前按 `checklist.md` 中的看板 P0 检查项逐条自检。

## 固定结构

- 页面背景：`填充-fill/0-透明`。
- 页面级顶部 Tab：用于看板大类、组织层级或一级视角切换，固定使用 `基础用法`。
- 页面顶部背景 Banner：页面背景视觉层，使用独立 `<img>` 展示 COS 原图，不作为内容容器，不加额外背景色、整层蒙层、滤镜或渐变。
- 正文内容层：从看板名称开始，统计时间、看板维度 Tab、首个内容卡片、第二个内容卡片和后续模块都放在同一个正文布局流中，按正常文档流从上到下排列。
- Banner 文案：看板名称必须显示，统计时间固定放在看板名称下方。
- 看板维度 Tab：存在指标口径、时间口径或对比方式切换时使用 `Banner 胶囊标签卡`。
- 首个内容卡片：属于正文内容层，优先放当前范围、核心指标、单位、同比/环比或趋势信息。
- 所有看板卡片：默认 `填充-fill/纯白`，不使用阴影，不使用描边。

## 背景与正文契约

- Banner 是页面顶部背景，不是布局容器；正文是唯一布局流。
- Banner 图片必须是独立 `<img>`，作为页面背景层显示在正文内容层下方，不能包裹标题、统计时间、Tab 或卡片。
- Banner 图片设置 `width: 100%`、`height: auto`，保持原图等比展示，不压缩、不拉伸、不裁剪。
- Banner 不参与正文布局计算，不决定标题、Tab、卡片的位置，也不作为首卡或其他模块的定位锚点。
- 正文内容层从看板名称开始，标题、统计时间、看板维度 Tab、首个内容卡片、第二个内容卡片和后续模块全部由同一个正常文档流定位。
- 有看板维度 Tab 时，首个内容卡片顶部与 看板维度 Tab 底部保持 20px。
- 没有看板维度 Tab 时，首个内容卡片顶部与统计时间标签底部保持 20px。
- 首个内容卡片与第二个内容卡片按普通看板卡片间距排列，优先使用 16px 或 20px。
- 所有正文纵向间距都是固定 px 或 Snow 间距 token，不随屏幕宽度、Banner 图片比例或页面宽度变化。
- 禁止为了调整首卡位置压缩 Banner 图片高度、裁剪 Banner 图片、改变图片比例，或给正文模块使用负 `margin-top`、`transform: translateY(...)`、`position: absolute/fixed`。
- Banner 背景层不得溢出覆盖正文内容可读性；正文层必须始终在 Banner 背景层之上。

推荐结构：

```html
<div class="dashboard-page">
  <img class="dashboard-bg-banner" src="..." alt="" />
  <main class="dashboard-content">
    <h1>看板名称</h1>
    <div class="dashboard-time">2025年2月</div>
    <div class="banner-tabs">...</div>
    <section class="dashboard-card dashboard-card--first">...</section>
    <section class="dashboard-card">...</section>
  </main>
</div>
```

推荐实现方式：

```css
.dashboard-page { position: relative; overflow: hidden; }
.dashboard-bg-banner { position: absolute; top: 0; left: 0; width: 100%; height: auto; z-index: 0; pointer-events: none; }
.dashboard-content { position: relative; z-index: 1; padding: 32px 16px 0; }
```

## 固定间距

- 看板标题字体 20-24px，标题距离 Banner 顶部 32px。
- 统计时间与标题间距 8px。
- 看板维度 Tab 与统计时间间距 20px。
- 有看板维度 Tab 时，首个内容卡片与 Tab 间距 20px。
- 没有看板维度 Tab 时，首个内容卡片与统计时间间距 20px。
- 首个内容卡片与第二个内容卡片间距优先使用 16px 或 20px。
- 上述纵向间距发生在正文内容层中，是固定 px，不随屏幕宽度、Banner 图片比例或页面宽度缩放。

## 响应式约束

- 适配 375/390/430px 宽度时，正文内容层的纵向间距保持一致。
- 适配只调整页面最大宽度、表格横向滚动和内容换行策略；Banner 背景图不裁剪、不压缩。
- 不要为了展示更多 Banner 图片而拉大标题、时间、Tab、首卡之间的距离。
- 不要通过压缩 Banner 图片高度、裁剪 Banner 图片或压缩 20px 间距制造首卡位置。
- Banner 图片底部弧形、波浪或凸出造型只作为背景视觉；不得覆盖或影响正文模块可读性。

## Banner 素材与展示

- Banner 图片从 `dashboard-banners.md` 的 COS 目录素材池获取。
- Banner 必须使用独立 `<img>` 节点作为页面顶部背景层展示，设置 `width: 100%`、`height: auto`。
- Banner 原图按屏幕宽度等比缩放，不压缩、不拉伸、不裁剪顶部、底部或左右边缘。
- 不要使用 CSS `background-image`、`background-size: cover`、`object-fit: cover` 或固定高度裁剪容器实现 Banner。
- `dashboard-banners.md` 中的示例图片只用于验证路径格式，不要固定只使用示例图片。
- 只有当正文内容可读性不足时，才允许在文字所在局部增加轻量文本阴影或局部弱遮罩。
- 局部弱遮罩不能覆盖整张 Banner，不能改变原图主色和层次。

## 强制图标

- 看板 KPI、数据指标卡、统计卡片必须包含 1 个指标图标。
- 排行榜、Top N、排名列表的第 1-3 名必须使用排名图标或排名徽章。
- 第 4 名以后使用普通数字序号。
- 没有真实图标资产时，使用 Remix Icon 或语义排名徽章占位，不能省略图标。

## 表格防溢出

- 看板表格必须放入横向滚动容器。
- 表格总宽可以大于屏幕，但页面本身不能横向溢出。
- 列数超过屏宽、字段较长或存在多指标对比时，保留横向滚动，不要强行压缩列宽。
- 数字、日期、百分比默认不换行；长文本单行省略或按业务允许换行。
- 固定首列时，首列与右侧滚动区行高必须一致。

## 禁止项

- 不要自创看板顶部结构。
- 不要自创组件规则中不存在的选项卡样式。
- 不要把页面级顶部 Tab 放进 Banner 背景层。
- 不要把 Banner 胶囊标签卡放进首个内容卡片内。
- 不要用文字、数字、业务缩写或 emoji 冒充图标。
- 不要让表格内容撑破页面边界。
