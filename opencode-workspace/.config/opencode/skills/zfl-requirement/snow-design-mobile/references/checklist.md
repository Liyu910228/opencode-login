# Snow Mobile 检查清单

交付前或做页面评审时使用本清单。

## 看板 P0 必查项

以下任一项不满足时，看板页视为不合格，必须先修复：

- Banner 只作为页面顶部背景层，不作为布局容器，没有包裹标题、统计时间、Tab 或卡片。
- Banner 背景图使用独立 `<img>` 展示 COS 素材池选中的原图，`width: 100%`、`height: auto`，未使用 `background-size: cover`、`object-fit: cover` 或固定高度裁剪容器。
- 看板标题、统计时间、看板维度 Tab、首个内容卡片都位于同一个正文内容层中，按固定 px 间距排布，间距不随屏幕宽度缩放。
- 有看板维度 Tab 时，首个内容卡片与 Tab 间距固定 20px；没有 Tab 时，首个内容卡片与统计时间间距固定 20px。
- 首个内容卡片通过正文内容层定位，未跨容器使用负 `margin-top`、`transform`、绝对定位或压缩背景图高度来调整位置。
- 看板 KPI、数据指标卡、统计卡片都包含 1 个指标图标。
- 排行榜、Top N、排名列表的第 1-3 名都使用排名图标或排名徽章。
- 看板表格放在横向滚动容器内，页面本身没有被表格撑出横向溢出。
- 看板页没有自创组件规则中不存在的选项卡样式。

## 产品适配

- 页面是 APP 功能页，不是营销页。
- 主要用户和核心任务清楚。
- 首屏包含有用的业务内容或主流程。
- 信息层级服务于业务扫读，而不是装饰效果。

## 信息架构

- 页面标题和当前上下文清楚。
- 顶部导航、页面内容、底部操作、反馈层级各司其职。
- 主要状态或关键指标容易发现。
- 相关字段分组合理。
- 必要操作可见，并按优先级排列。
- 页面内切换、跨页面跳转、一级模块导航没有混用。
- 相关的空、加载、错误、禁用、无权限状态已处理。

## Snow Mobile 一致性

- 字体使用 Snow 字号阶梯和平台/系统字体规则。
- 颜色使用 Snow 语义 token，不使用随意色值。
- 间距使用 Snow 间距 token，不混用零散数值。
- 圆角使用 Snow 圆角 token，容器类和控件类层级稳定。
- 页面模式符合详情、看板、列表、表单或结果页的用法。
- 看板页整体页面背景使用 `填充-fill/0-透明` 浅灰色，未使用纯白作为整页背景。
- 看板页首屏包含顶部导航、Banner 背景图、看板名称、统计时间和首个内容卡片。
- 看板 Banner 是页面顶部背景层，宽度等于移动端页面宽度，没有作为布局容器包裹正文内容。
- 看板标题字体 20-24px，距离 Banner 顶部 32px；统计时间固定放在看板名称下方，与标题间距 8px。
- 有看板维度 Tab 时，Tab 与统计时间间距 20px，首个内容卡片与 Tab 间距 20px；没有 Tab 时，首个内容卡片与统计时间间距 20px。
- 看板页页面级顶部 Tab 位于顶部导航栏下方、正文内容上方，使用基础用法；看板维度 Tab 位于正文内容层中，使用 Banner 胶囊标签卡。
- 看板页没有把页面级顶部 Tab 放进 Banner 背景层，也没有把看板维度 Tab 下移到首个内容卡片内部。
- 看板维度 Tab 的每个选项都是独立胶囊，选项间距固定 8px，没有合并成一整条半透明底板。
- Banner 胶囊标签卡的未选中项在 页面顶部 Banner 背景上清晰可读，没有因透明度过低而消失。
- Banner 原图作为页面顶部背景图按屏幕宽度等比展示，顶部、底部和左右边缘未被裁剪、压缩或拉伸。
- 首个内容卡片属于正文内容层，不放进 Banner 图片节点，不遮挡看板名称、统计时间和看板维度 Tab。
- 首个内容卡片未通过负 `margin-top`、`transform: translateY(...)`、`position: absolute/fixed` 或压缩 Banner 背景图高度来调整位置。
- 看板页所有卡片默认使用 `填充-fill/纯白` 白色背景，且不使用阴影、不使用描边，包括首个内容卡片、KPI 卡片、风险提醒卡片、排行卡片、趋势卡片和明细卡片。
- 看板 Banner 背景图已从 COS 目录素材池读取图片列表并随机选图，未固定只使用示例图片。
- 看板 Banner 背景图直接用独立 `<img>` 展示选中的原图，未添加额外背景色，未叠加整层蓝色、渐变、半透明蒙层、模糊层或滤镜。
- 按钮遵循主/次/危险操作的强调规则。
- 图标符合 Remix Icon 或 AI 生成两类资产渠道。
- PRD 未描述图标时，已按页面结构判断应加和不应加图标的位置。
- 首页/工作台、看板 KPI、底部 Tab、搜索框、工具操作等结构性图标位已完成盘点。
- 页面中没有用单个汉字、业务缩写、首字母、数字或 emoji 冒充图标。
- 首页/工作台功能入口使用面性彩色图标，低保真占位已明确标注。
- 底部 Tab 使用图标加文案，不使用文字占位图标。
- 搜索框包含搜索图标。
- 看板数据指标图标使用浅色圆角方底，图标主体无额外底色。
- 每个看板 KPI、数据指标卡、统计卡片都包含 1 个指标图标；缺少图标视为不合格。
- 浅色圆角方底图标的底色与图标主体为同一色系，没有橙色图标配浅绿底等错配。
- 排行榜、Top N、排名列表的第 1-3 名使用排名图标或排名徽章，第 4 名以后使用普通数字序号。
- 徽标表达真实状态或数量。
- 表格已根据业务选择基础表格、固定首列、横向滚动、分组表头、排序、帮助、状态、下钻等能力。
- 表格、表头和分组表头使用固定基础视觉：白色表格背景、浅品牌表头背景、统一字体、弱分隔线和一致图标颜色。
- 表格列宽、行高和表头高度在移动端可读范围内，没有为塞满一屏强行压缩。
- 表格外层使用横向滚动容器，表格内容没有撑破页面宽度，也没有溢出单元格边界。
- 数字、日期、百分比默认不换行；长文本按规则省略或业务允许时换行。
- 分组表头只用于真实字段层级，一级表头横跨子列，子列表头与数据列对齐。
- 数字列右对齐，文本列左对齐，同一列对齐方式一致。
- 选项卡只用于同一上下文中的同级分组。
- 看板维度 Tab 使用 Banner 胶囊标签卡，选项间距固定 8px，单项宽度按容器可用宽度等分计算。
- 看板正文模块内的局部切换使用组件规则中已有的选项卡变体，没有自创灰底白块、半透明底板等未定义样式。
- 同一页面中，同一层级、同一交互目的的选项卡样式一致。
- 分页/加载更多只在列表确实需要时使用。

## 移动端可用性

- 文字能放进容器，不重叠、不异常截断。
- 点击目标适合移动端操作。
- 底部固定操作不遮挡关键内容。
- 搜索、筛选、排序、选项卡状态不会互相误清空。
- 列表使用真实业务数据时仍然可扫读。
- 关键数字和状态不被截断。

## 评审输出格式

做评审时按以下方式输出：

1. 高影响问题优先。
2. 指明具体位置或组件。
3. 说明违反了哪条规范。
4. 给出具体修复建议。

避免只说“优化布局”“更美观”这类没有明确改法的反馈。
