# 销量快报 功能需求方案文档

> **版本**：v1.1
> **日期**：2026-06-24
> **适用系统**：华润雪花营销驾驶舱 H5
> **需求来源**：王乾《销量快报需求说明》
> **优先级**：分档次/分品牌/分主力品种/其他品种（均为 P0）

> **页面视觉规格**：`./ui-spec.md`
> **demo链接**：`./demo.html`

## 1. 需求背景与目标

### 1.1 项目背景

华润雪花啤酒营销驾驶舱 H5 是面向内部销售、业务部、营销中心及管理层的移动端数据看板系统。现有看板已覆盖库存、经营分析、风险监控等模块。本次新增"销量快报"模块，旨在让各级管理人员能够快速、直观地查看不同维度（档次、品牌、品种）的销量生意表现，辅助业务决策。

### 1.2 业务目标

- 提供统一的销量快报入口，减少多系统切换成本
- 支持按多维度（档次/品牌/品种）快速切换查看销量数据
- 支持组织层级切换（事业部/营销中心），实现横向对比
- 通过汇总卡片 + 条形图 + 数据表格的组合形式，在移动端高效展示核心指标

### 1.3 用户目标

- **事业部管理层**：快速掌握各档次/品牌/品种整体销量表现，识别增长/下滑维度
- **营销中心人员**：查看所辖区域组织排名，横向对比各组织表现
- **业务部人员**：查看具体品种销量占比和增幅，精细化运营

### 1.4 本轮要解决的问题

1. 缺少统一的销量快报查看入口，数据分散
2. 多维度销量数据无法在一个页面内快速切换对比
3. 组织层级切换不便，难以横向对比
4. 缺少品种维度的销量占比可视化展示

### 1.5 不在本轮解决的问题

- 数据导出功能
- 折线图/柱状图等复杂图表可视化
- 数据下钻到具体客户/门店
- 实时数据刷新/推送
- 多组织层级同时对比

>
> 用户进入销量快报 → 默认"分档次"Tab，子维度默认"分档次生意表现" → 横滑选择子维度（如"普高及以上生意表现"）→ 内容区展示对应汇总卡片 → 组织切换（分事业部/分营销中心）→ 查看数据表格 → 可切换其他 Tab 或使用筛选/时间功能

## 3. 需求详情

### 3.1 顶部页签切换

#### 3.1.1 适用角色与权限

- **全部角色**：均可查看

#### 3.1.2 触发入口

**入口 1 — 营销驾驶舱首页（本期主入口）**：
- 营销驾驶舱 H5 首页「销量快报」入口

**入口 2 — 直接链接（辅助入口）**：
- 通过分享链接或扫码直接进入

#### 3.1.3 交互步骤

1. 用户进入销量快报页面
2. 默认选中「分档次」Tab，子维度默认「分档次生意表现」
3. 用户可点击其他 Tab 切换大维度
4. 切换 Tab 后，Banner、子维度导航、汇总卡片、数据表格同步更新

#### 3.1.4 业务规则

- Tab 切换时保留当前组织页签（分事业部/分营销中心）和时间筛选状态
- Tab 切换后，子维度导航重置为第一项（总览）
- 分品牌 Tab 无筛选按钮，其他 Tab 有筛选按钮

#### 3.1.5 开发必看

> Tab 切换时，组织页签和时间筛选状态需要保留，避免用户重复操作。子维度导航需重置。

#### 3.1.6 重点注意

> Tab 切换后，Banner 标题、汇总卡片指标、表格字段都会变化，需要确保数据一致性。

### 3.2 子维度横滑导航

#### 3.2.1 适用角色与权限

- **全部角色**：均可查看

#### 3.2.2 子维度清单

| Tab | 子维度列表 |
|-----|-----------|
| 分档次 | 分档次生意表现、普高及以上生意表现、次高生意表现、中档生意表现、大主流生意表现 |
| 分品牌 | 分品牌生意表现（单一，无横滑导航） |
| 分主力品种 | 分主力品种生意表现、喜力生意表现、纯生生意表现、superX生意表现、老雪生意表现、红爵生意表现、勇闯系列生意表现 |
| 其他品种 | 其他品种生意表现、苏尔生意表现、黑狮生意表现、红爵轻卡生意表现、马尔斯绿生意表现、雪花脸谱生意表现、秦始皇生意表现 |

#### 3.2.3 交互步骤

1. 用户在子维度横滑列表中点击具体维度
2. 内容区切换为对应维度的展示内容
3. 总览维度（第 1 项）展示汇总对比信息，单品维度（第 2+ 项）展示具体生意表现

#### 3.2.4 业务规则

- 总览维度：展示全维度对比（档次结构卡片 / 条形图 / 品种生意表现表）
- 单品维度：展示该维度的汇总卡片 + 组织切换 + 数据表格
- 切换子维度时保留组织页签和时间筛选状态

#### 3.2.5 开发必看

> 子维度导航是横滑标签卡，超过屏幕宽度时横向滚动。分品牌 Tab 只有 1 个维度，不需要横滑导航。

### 3.3 Banner 横幅

#### 3.3.1 页面与视图

- Banner 主标题：随 Tab 动态变化
  - 分档次：分档次生意表现

#### 3.3.3 重点注意

> 分品牌 Tab 没有筛选按钮，其他 Tab 有筛选按钮。筛选弹窗需实现完整的单选交互。

### 3.8 数据表格（通用结构）

#### 3.8.1 表头

组织排名 | 当期 | 同期 | 同比增减 | 同比增幅 | 增量贡献（进度条）

#### 3.8.2 业务规则

- 组织排名按当期销量降序排列
- 同比增幅用颜色区分正负
- 增量贡献用进度条展示，进度条长度按占比计算
- 固定首列（组织名称），支持横向滚动

#### 3.8.3 开发必看

> 表格列数较多（6 列），移动端需横向滚动。固定首列（组织名称 + 排名），其他列可滚动。

#### 3.8.4 重点注意

> 进度条展示规则需后端确认：增量贡献的计算口径是什么？按什么维度计算占比？

## 4. 功能开发事项

### 4.1 前端开发事项

| 事项 | 说明 |
|------|------|
| 顶部 Tab 切换组件 | 实现页面级 Tab 切换（基础用法），保留状态 |

| 档次结构卡片 | 4 行对比表，占比差异颜色区分 |

| 组织切换组件 | 分段器：分事业部/分营销中心 |
| 表格组件 | 固定首列，支持横向滚动，进度条展示 |

| 时间选择组件 | 底部弹出时间选择器 |
| 空状态/加载状态 | 加载动画、暂无数据、网络异常 |

| 事项 | 说明 |
|------|------|

## 6. 风险与处理策略

| 风险 | 影响 | 处理策略 |
|------|------|----------|
| **需求理解风险**：进度条计算口径不明确 | 表格展示 | 与后端确认增量贡献的计算规则 |
| **交互风险**：子维度+组织切换+筛选多重状态管理 | 状态一致性 | 设计清晰的状态管理方案 |
| **技术依赖风险**：数据接口延迟 | 用户体验 | 增加加载动画，超时提示 |
| **数据一致性风险**：多维度数据口径不一致 | 数据准确性 | 与后端确认各维度数据口径 |
| **性能风险**：子维度切换频繁请求 | 性能 | 做防抖处理，数据缓存 |
| **排期风险**：需求变更 | 项目进度 | 严格控制需求变更，变更需评审 |

## 7. 非功能需求

| 维度 | 要求 |
|------|------|
| **性能** | 页面加载 < 2s，数据查询 < 1s，接口超时 5s |
| **安全** | 数据按组织层级隔离，敏感数据加密传输 |
| **稳定性** | 可用性 > 99.5%，异常时友好提示 |
| **易用性** | 关键指标一目了然，子维度横滑快速切换 |
| **可维护性** | 组件化开发，4 个 Tab 结构可复用 |
| **兼容性** | 适配 iOS 12+/Android 8+，主流机型 |
| **可观测性** | 关键操作埋点，便于分析用户行为 |
