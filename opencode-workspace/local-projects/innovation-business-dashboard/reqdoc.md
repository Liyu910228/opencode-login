# 创新业务看板 功能需求方案文档

> **版本**：v1.0
> **日期**：2026-07-01
> **适用系统**：华润雪花创新业务移动端看板
> **需求来源**：管理层移动端数据查看需求
> **优先级**：整体页签（P0）、代工类页签（P0）、日/月/年切换（P0）、组织维度切换（P0）、下钻一级（P0）

> **页面视觉规格**：`innovation-business-dashboard/ui-spec.md`
> **demo链接**：`innovation-business-dashboard/demo.html`

---

## 1. 需求背景与目标

### 1.1 项目背景

华润雪花创新业务涵盖代工类、特色产品、桶啤三大板块。当前管理层缺少统一的移动端数据查看入口，无法快速了解各业务分类的销量表现和同比变化。需要建设一个移动端 H5 看板，支持按业务分类、时间维度和组织层级查看销量数据。

### 1.2 业务目标

- 管理层可在手机上随时查看创新业务各分类的销量数据
- 直观展示当期、同期、同比增减、同比增幅，辅助经营决策
- 支持按组织层级下钻，快速定位各层级排名情况

### 1.3 用户目标

- **总部管理层**：查看全国创新业务整体销量，按事业部/营销中心维度查看排名，下钻到事业部或营销中心明细
- **事业部管理层**：查看本事业部销量及全国各事业部对比，下钻到营销中心明细
- **营销中心管理层**：查看本营销中心销量及全国营销中心对比，下钻到业务部明细
- **销售大区管理层**：查看本大区销量，下钻到业务部明细

### 1.4 本轮要解决的问题

1. 移动端看板整体/代工类/特色产品/桶啤四大分类的数据展示
2. 日/月/年三种时间维度切换
3. 组织维度（事业部/营销中心）排名查看
4. 下钻一级查看更细分组织数据

### 1.5 不在本轮解决的问题

- 特色产品、桶啤一级页签的子主题看板结构（暂定）
- PC 端看板
- 数据导出功能
- 手动选择具体日期/月份/年份

## 2. 需求范围

### 2.1 需求清单

| 需求名称 | 优先级 | 适用角色 | 关键用户旅程 | 是否本期必做 |
|----------|--------|----------|--------------|--------------|
| 整体一级页签（含4个主题看板） | P0 | 全部角色 | 进入看板→查看整体汇总→切换主题看板→切换组织维度→下钻查看明细 | 是 |
| 代工类一级页签（含4个主题看板） | P0 | 全部角色 | 切换到代工类→查看汇总→切换代加工/包销/定制主题→下钻查看明细 | 是 |
| 日/月/年时间维度切换 | P0 | 全部角色 | 点击日/月/年Tab切换时间维度 | 是 |
| 组织维度切换（事业部/营销中心） | P0 | 全部角色 | 在排名表格区域切换事业部/营销中心维度 | 是 |
| 下钻一级（底部弹窗） | P0 | 全部角色 | 点击排名表格某行→底部弹窗展示更细分组织数据 | 是 |
| 特色产品一级页签 | P1 | 全部角色 | 暂定 | 否 |
| 桶啤一级页签 | P1 | 全部角色 | 暂定 | 否 |

### 2.2 核心流程

> 流程概述：
> 1. 用户进入看板页面，默认展示"整体"一级页签，默认展示"整体主题看板"
> 2. 页面顶部显示一级页签导航：整体 | 代工类 | 特色产品 | 桶啤
> 3. 一级页签下方显示时间维度 Tab：日 | 月 | 年（默认月）
> 4. Banner 区域显示看板名称和导航按钮，点击导航按钮侧边展开选择列表，可切换当前一级页签下的主题看板
> 5. 主体内容区展示：全国汇总卡片 + 组织排名表格
> 6. 组织排名表格支持"事业部/营销中心"维度 Tab 切换
> 7. 点击表格行触发下钻，底部弹窗展示更细分的组织数据
> 8. 不同角色看到的数据范围受权限控制

## 3. 需求详情

### 3.1 整体一级页签

#### 3.1.1 适用角色与权限

- **总部角色**：看全部事业部和全部营销中心数据，可下钻到事业部，再下钻到营销中心
- **事业部角色**：看本事业部下全部营销中心 + 全国各事业部数据，可下钻到营销中心
- **营销中心角色**：看本营销中心下全部业务部 + 全国营销中心，可下钻到业务部
- **销售大区角色**：看本大区下全部业务部，可下钻到业务部

#### 3.1.2 触发入口

**入口 1 — 看板首页（唯一入口）**：
- 用户从 APP 首页或工作台进入创新业务看板，默认展示"整体"一级页签

#### 3.1.3 页面与视图

> **页面视觉规格**：参考 `ui-spec.md`
> **界面原型**：参考 `demo.html` → 默认展示整体页签。以下为关键界面说明：

**整体页签 - 整体主题看板**：

- 页面标题：创新业务看板
- 一级页签：整体 | 代工类 | 特色产品 | 桶啤
- 时间维度 Tab：日 | 月 | 年（默认月）
- Banner 导航按钮：整体看板 | 代工类主题 | 特色产品主题 | 桶啤主题
- 全国汇总卡片：全国销量（KL）及同比增减、同比增幅
- 业务分类汇总：代工类、特色产品、桶啤三类业务的当期、同期、同比增减、同比增幅
- 组织排名表格：按事业部/营销中心维度切换，展示组织名称、当期、同期、同比增减、同比增幅

**整体页签 - 代工类/特色产品/桶啤主题看板**：

- 仅展示该主题的全国汇总和不同组织的明细表
- 结构同整体主题看板的汇总部分

#### 3.1.4 交互步骤

1. 用户进入看板，默认展示"整体"一级页签 + "整体主题看板"
2. 页面展示全国汇总卡片（销量、同比增减、同比增幅）
3. 展示代工类/特色产品/桶啤三类业务的当期、同期、同比增减、同比增幅
4. 展示组织排名表格，默认"事业部"维度
5. 用户可切换"营销中心"维度查看排名
6. 用户点击 Banner 导航按钮，侧边展开选择列表，选择切换主题看板
7. 用户点击表格某行，底部弹窗展示下钻数据

#### 3.1.5 关键状态

| 状态 | 说明 / 展示方式 |
|------|----------------|
| 同比增减为正 | 绿色数字，前面带 + 号 |
| 同比增减为负 | 红色数字 |
| 同比增减为 0 | 灰色 0 |
| 同期为 0 | 同比增幅显示 "-" |
| 加载中 | 骨架屏 |
| 无数据 | 空状态提示 |

#### 3.1.6 业务规则

- **同比增减** = 当期 - 同期
- **同比增幅** = 同比增减 / 同期 × 100%，最多保留 1 位小数
- **同期为 0 时**：同比增幅显示 "-"
- **默认时间维度**：月
- **不支持手动选择具体时间**，只看当期默认数据
- **组织排名**：按当期销量降序排列
- **下钻规则**：总部角色从事业部下钻到营销中心；事业部角色从营销中心下钻（不可再钻）；营销中心角色从业务部下钻（不可再钻）

#### 3.1.7 数据输入输出

- **输入**：一级页签类型、时间维度（日/月/年）、主题看板类型、组织维度（事业部/营销中心）、下钻组织 ID
- **输出**：全国汇总数据、业务分类汇总数据、组织排名列表、下钻明细数据

#### 3.1.8 异常或边界情况

- **同期为 0**：同比增幅显示 "-"，同比增减正常显示
- **当期和同期都为 0**：同比增减显示 0，同比增幅显示 "-"
- **无下钻数据**：点击行不触发弹窗，或弹窗显示"暂无下级数据"
- **权限不足**：显示"暂无权限查看"提示

#### 3.1.9 开发必看

> 1. 整体页签下包含 4 个主题看板：整体看板（默认）、代工类主题、特色产品主题、桶啤主题
> 2. 主题看板切换通过 Banner 内导航按钮实现，点击后侧边展开选择列表
> 3. 整体主题看板展示：全国汇总 + 三类业务汇总 + 组织排名表格
> 4. 其他三个主题看板仅展示：该主题全国汇总 + 组织排名表格
> 5. 组织排名表格只有"事业部"和"营销中心"两个维度 Tab
> 6. 下钻通过底部弹窗实现，弹窗内展示更细分的组织数据

#### 3.1.10 重点注意

> 1. 同比增幅 = 同比增减 / 同期，分母为 0 时必须处理
> 2. 不同角色的数据范围和可下钻层级不同，必须严格按权限控制
> 3. 日维度的同期对比逻辑需确认（当天 vs 去年同日）
> 4. 特色产品和桶啤主题看板结构暂定，本期按简单汇总展示

#### 3.1.11 风险提醒

> 1. 同期为 0 的计算边界需要前后端统一处理逻辑
> 2. 下钻层级的权限控制需要与后端接口配合，避免越权查看
> 3. 日维度的同期对比逻辑未确认，可能影响数据准确性

### 3.3 日/月/年时间维度切换

#### 3.3.1 适用角色与权限

全部角色。

#### 3.3.2 触发入口

**入口 1 — 时间维度 Tab（页面级）**：
- 位于一级页签下方，全局可见

#### 3.3.3 页面与视图

- 使用分段器组件：日 | 月 | 年
- 默认选中"月"
- 切换后刷新当前页面数据

#### 3.3.4 交互步骤

1. 用户点击"日"/"月"/"年"Tab
2. 选中态切换，数据刷新
3. 页面展示对应时间维度的数据

#### 3.3.5 关键状态

| 状态 | 说明 |
|------|------|
| 日 | 展示当天数据，同期为去年同日 |
| 月 | 展示当月数据，同期为去年同月 |
| 年 | 展示当年数据，同期为去年 |

#### 3.3.6 业务规则

- 默认选中"月"
- 不支持手动选择具体日期/月份/年份
- 切换时间维度后，当前一级页签和主题看板保持不变

#### 3.3.7 数据输入输出

- **输入**：时间维度（日/月/年）
- **输出**：对应时间维度的当期、同期数据

#### 3.3.8 异常或边界情况

- **日维度跨年**：当天为 1 月 1 日时，同期为去年 1 月 1 日
- **月维度**：当月无数据时显示空状态

#### 3.3.9 开发必看

> 1. 时间维度切换是全局的，影响当前一级页签下的所有主题看板
> 2. 切换后需要重新请求数据接口
> 3. 日维度的同期对比逻辑需与后端确认

#### 3.3.10 重点注意

> 1. 不支持手动选择具体时间，只看默认当期
> 2. 切换时间维度后，组织维度和下钻状态应重置

#### 3.3.11 风险提醒

> 1. 日维度的同期对比逻辑未确认，可能影响数据准确性

### 3.4 组织维度切换

#### 3.4.1 适用角色与权限

全部角色。

#### 3.4.2 触发入口

**入口 1 — 组织维度 Tab（表格区域）**：
- 位于组织排名表格上方

#### 3.4.3 页面与视图

- 使用分段器或标签卡组件：事业部 | 营销中心
- 默认选中"事业部"
- 切换后刷新排名表格数据

#### 3.4.4 交互步骤

1. 用户点击"事业部"/"营销中心"Tab
2. 选中态切换，排名表格刷新
3. 展示对应维度的组织排名

#### 3.4.5 关键状态

| 状态 | 说明 |
|------|------|
| 事业部 | 展示各事业部排名 |
| 营销中心 | 展示各营销中心排名 |

#### 3.4.6 业务规则

- 只有"事业部"和"营销中心"两个维度
- 不同角色看到的数据范围不同，但维度切换按钮一致
- 切换维度后，下钻状态重置

#### 3.4.7 数据输入输出

- **输入**：组织维度（事业部/营销中心）
- **输出**：对应维度的组织排名列表

#### 3.4.8 异常或边界情况

- **无数据**：显示空状态

#### 3.4.9 开发必看

> 1. 组织维度切换只影响排名表格，不影响汇总卡片
> 2. 切换维度后需要重新请求排名数据接口

#### 3.4.10 重点注意

> 1. 只有事业部/营销中心两个维度，不要做成更多维度
> 2. 切换维度后，之前的下钻弹窗应关闭

#### 3.4.11 风险提醒

> 无明显风险。

### 3.5 下钻一级

#### 3.5.1 适用角色与权限

同 3.1.1，不同角色可下钻的层级不同。

#### 3.5.2 触发入口

**入口 1 — 排名表格行点击（唯一入口）**：
- 点击排名表格中的某一行，触发下钻

#### 3.5.3 页面与视图

- 使用底部弹窗（Bottom Sheet）展示下钻数据
- 弹窗标题显示当前组织名称和下级组织维度
- 弹窗内展示更细分的组织排名列表
- 弹窗可关闭

#### 3.5.4 交互步骤

1. 用户点击排名表格某行
2. 底部弹窗弹出，展示该组织的下级组织排名
3. 弹窗内展示下级组织名称、当期、同期、同比增减、同比增幅
4. 用户可关闭弹窗返回

#### 3.5.5 关键状态

| 状态 | 说明 |
|------|------|
| 弹窗打开 | 展示下级组织排名 |
| 弹窗关闭 | 返回主页面 |
| 无下级数据 | 弹窗内显示"暂无下级数据" |

#### 3.5.6 业务规则

- **总部角色**：事业部维度下钻到营销中心，营销中心维度不可再钻
- **事业部角色**：营销中心维度下钻（不可再钻）
- **营销中心角色**：业务部维度下钻（不可再钻）
- **销售大区角色**：业务部维度下钻（不可再钻）
- 下钻只支持一级，不支持多级连续下钻
- 弹窗内数据按当期销量降序排列

#### 3.5.7 数据输入输出

- **输入**：当前组织 ID、当前组织维度、下级组织维度
- **输出**：下级组织排名列表

#### 3.5.8 异常或边界情况

- **无下级数据**：弹窗显示"暂无下级数据"
- **权限不足**：弹窗显示"暂无权限查看"

#### 3.5.9 开发必看

> 1. 下钻通过底部弹窗实现，不是页面跳转
> 2. 不同角色的下钻目标层级不同，需要按角色判断
> 3. 弹窗内不再支持下钻（只支持一级）
> 4. 弹窗关闭后，主页面的维度和数据状态保持不变

#### 3.5.10 重点注意

> 1. 下钻只支持一级，不要做成多级连续下钻
> 2. 弹窗内的数据字段与主表格一致
> 3. 弹窗需要适配不同屏幕高度

#### 3.5.11 风险提醒

> 1. 弹窗高度需要适配不同屏幕，避免内容被遮挡
> 2. 下钻权限控制需要与后端接口配合

## 4. 功能开发事项

### 4.1 前端开发事项

| 事项 | 说明 |
|------|------|
| 一级页签切换 | 整体/代工类/特色产品/桶啤，使用基础用法选项卡 |
| 时间维度切换 | 日/月/年，使用分段器组件，默认月 |
| Banner 导航按钮 | 点击侧边展开选择列表，切换主题看板 |
| 组织维度切换 | 事业部/营销中心，使用分段器或标签卡 |

## 6. 风险与处理策略

| 风险 | 影响 | 处理策略 |
|------|------|----------|
| **需求理解风险**：日维度同期对比逻辑未确认 | 数据准确性 | 与业务方确认后补充 |
| **交互风险**：下钻层级与角色权限映射复杂 | 越权查看 | 前后端双重校验权限 |
| **技术风险**：同期为0时同比增幅计算 | 展示异常 | 后端统一处理，前端展示"-" |
| **数据风险**：特色产品/桶啤子主题结构暂定 | 后续返工 | 本期按简单汇总展示，后续迭代 |
| **排期风险**：4个一级页签+多个主题看板 | 开发量大 | 优先完成整体和代工类，特色产品/桶啤暂定 |

## 7. 非功能需求

| 维度 | 要求 |
|------|------|
| **性能** | 页面加载时间 < 3 秒，数据接口响应 < 1 秒 |
| **安全** | 按角色控制数据可见范围，防止越权查看 |
| **稳定性** | 同期为0等边界情况不报错，正常展示 |
| **易用性** | 移动端适配 375/390/430px，表格横向滚动不溢出 |
| **可维护性** | 组件化开发，主题看板结构可复用 |
| **兼容性** | 支持主流移动端浏览器（Safari、Chrome、微信内置浏览器） |
