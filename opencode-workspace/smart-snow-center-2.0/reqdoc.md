# 智慧雪花中心 2.0 功能需求方案文档

> **版本**：v1.0
> **日期**：2026-07-03
> **适用系统**：智慧雪花中心 2.0（Smart Snow Center 2.0）
> **需求来源**：用户提供智慧雪花中心 2.0 总部看板截图
> **优先级**：总部看板（P0）、报表卡片组件（P0）、左侧导航（P0）、搜索功能（P0）、推荐系统（P1）

> **demo链接**：`smart-snow-center-2.0/demo.html`

---

## 1. 需求背景与目标

### 1.1 项目背景

华润啤酒（雪花啤酒）内部已建设「智慧雪花中心 1.0」，作为 BI 报表和数据看板的聚合门户，为总部及区域/片区的销售管理、业务运营人员提供统一的数据查看入口。

随着业务发展和用户体验要求的提升，现有 1.0 版本在界面视觉、交互体验、信息架构等方面需要全面升级，推出「智慧雪花中心 2.0」。2.0 版本的核心目标是通过 UI/UX 全面改版，提升平台的易用性、信息获取效率和视觉品质。

### 1.2 业务目标

- 提升报表查找效率，通过分类导航和搜索功能让用户快速定位所需报表
- 统一报表卡片设计规范，建立一致的视觉语言和操作体验
- 引入推荐系统，引导用户使用 BI 自助分析、智能问数等高级数据能力
- 保持 1.0 到 2.0 的平滑过渡，降低用户迁移成本

### 1.3 用户目标

- **总部管理层**：一屏掌握各业务板块核心报表入口，快速查看经营数据
- **总部业务分析人员**：高效搜索和收藏常用报表，建立个人工作台
- **区域/片区管理人员**：查看本区域相关报表，对标其他区域表现

### 1.4 本轮要解决的问题

1. 1.0 版本界面陈旧，信息架构不够清晰，需要全面 UI/UX 升级
2. 报表入口分散，缺乏统一的卡片规范和操作反馈
3. 缺少智能推荐和搜索能力，用户找报表效率低
4. 需要支持 1.0 到 2.0 的过渡期兼容

### 1.5 不在本轮解决的问题

- 智能问数功能的具体实现（仅作为推荐入口）
- BI营销自助分析平台的具体功能（仅作为推荐入口）
- 报表卡片内数据的实时预览
- 个性化推荐算法的具体实现

## 3. 需求详情

### 3.1 整体布局框架

#### 3.1.1 适用角色与权限

- **全部用户**：总部管理层、总部业务分析人员、区域/片区管理人员、一线业务人员
- **权限说明**：所有用户均可查看总部看板页面；各业务板块的报表数据范围按用户所属组织层级隔离（待确认具体权限规则）

#### 3.1.2 触发入口

**入口 1 — 登录后默认进入（本期主入口）**：
- 用户登录智慧雪花中心 2.0 后，默认进入总部看板页面

**入口 2 — 左侧导航切换（辅助入口）**：
- 用户点击左侧导航菜单项，切换到对应业务板块页面

#### 3.1.3 页面与视图

> **界面原型**：参考 `demo.html` → 按原型中的页面入口查看。以下为关键界面说明：

**整体布局**：

![智慧雪花中心2.0-整体布局](images/智慧雪花中心2.0-整体布局.png)

- **左侧导航栏**：固定宽度，包含菜单树（我的工作台、总部看板、各业务板块等）
- **顶部功能栏**：固定高度，包含 Logo、系统名称、功能按钮、用户信息
- **主内容区**：自适应宽度，展示当前选中菜单对应的页面内容
- **背景**：浅蓝渐变背景（从截图可见）

**布局规格**：
- 左侧导航栏宽度：约 200px（待确认精确值）
- 顶部功能栏高度：约 60px（待确认精确值）
- 主内容区：剩余空间，带内边距

#### 3.1.4 交互步骤

1. 用户登录系统，页面加载总部看板
2. 左侧导航默认展开「总部看板」菜单项（高亮选中状态）
3. 主内容区展示推荐系统、搜索框、分板块报表卡片
4. 用户点击左侧导航其他菜单项，主内容区切换为对应板块内容
5. 用户点击「我的工作台」，展开子菜单（最近访问、我的收藏）

#### 3.1.5 关键状态

| 状态 | 说明 / 展示方式 |
|------|----------------|
| 菜单选中态 | 左侧导航当前菜单项高亮（蓝色背景） |
| 菜单展开态 | 「我的工作台」可展开显示子菜单 |
| 页面加载态 | 主内容区显示加载动画 |
| 空数据态 | 某板块无报表时显示空状态提示 |

#### 3.1.6 业务规则

- **导航层级**：支持两级菜单（父菜单 + 子菜单）
- **菜单选中**：同一时间仅一个菜单项处于选中状态
- **页面切换**：切换菜单时主内容区无刷新切换（SPA 模式）或整页跳转（待确认）

#### 3.1.7 数据输入输出

- **输入**：用户点击菜单项
- **输出**：主内容区展示对应板块页面

#### 3.1.8 异常或边界情况

- **菜单数据加载失败**：左侧导航显示默认菜单结构或错误提示
- **页面加载超时**：主内容区显示加载失败提示和重试按钮

#### 3.1.9 开发必看

> - 左侧导航菜单结构需要支持动态配置（后端返回菜单数据），便于后续新增业务板块
> - 菜单选中状态需要与路由/页面状态同步
> - 「我的工作台」为可展开父菜单，其他菜单项为叶子节点（从截图推断）

#### 3.1.10 重点注意

> - 左侧导航的「总部看板」当前为选中状态，说明这是默认首页
> - 「片区|事业部定制」和「统计报表」带有箭头图标，暗示可能有子菜单或独立页面

#### 3.1.11 风险提醒

> - 菜单结构如果硬编码在前端，后续新增业务板块需要发版；建议采用后端配置化方案

#### 3.5.6 业务规则

- **收藏**：用户可收藏报表，收藏后在「我的收藏」中快速访问
- **点赞**：用户可点赞报表，点赞数反映报表受欢迎程度
- **浏览**：用户每次进入报表详情页，浏览量+1
- **分享**：用户可分享报表给其他用户（分享方式待确认）
- **编辑**：仅特定角色可编辑报表配置（待确认权限规则）
- **设备标识**：卡片上的手机/电脑图标表示该报表支持的终端类型

#### 3.5.7 数据输入输出

- **输入**：用户操作（点击卡片、收藏、点赞、分享、编辑）
- **输出**：
  - 点击卡片：跳转到报表详情页
  - 收藏/点赞：更新用户个人数据，更新卡片显示数字
  - 分享：生成分享链接或二维码
  - 编辑：进入报表编辑页面

#### 3.5.8 异常或边界情况

- **报表已下架**：卡片置灰或显示「已下架」标签，点击提示不可用
- **操作失败**：收藏/点赞操作失败时显示 toast 提示
- **权限不足**：无编辑权限的用户点击编辑按钮提示权限不足

#### 3.5.9 开发必看

> - 报表卡片需要组件化封装，便于在各业务板块页面复用
> - 卡片上的操作数字（收藏、点赞、浏览）需要实时从后端获取
> - 收藏状态需要与用户个人数据同步，刷新页面后保持收藏状态
> - 设备标识图标需要根据报表配置的终端类型动态显示

#### 3.5.10 重点注意

> - 从截图看，「次高战略日报」「大湾区日报」「创新业务」的收藏星形为实心（橙色），说明这些报表已被当前用户（王乾）收藏
> - 不同报表的图标颜色不同，可能代表不同业务板块或报表类型（待确认配色规则）
> - 「市场新鲜度(试行)」标题带「(试行)」标记，需要在卡片上特殊展示

#### 3.5.11 风险提醒

> - 收藏、点赞、浏览等操作需要后端接口支持，接口需要防刷机制
> - 如果报表数量较多，卡片列表需要分页或虚拟滚动

## 6. 风险与处理策略

| 风险 | 影响 | 处理策略 |
|------|------|----------|
| **需求理解风险**：仅有一张总部看板截图，其他页面内容未知 | 可能导致需求遗漏或与实际情况不符 | 尽快补充各业务板块页面的截图或说明；当前文档基于截图推断，标注待确认项 |
| **交互或流程风险**：部分交互细节不明确（如搜索结果展示方式、分享方式等） | 可能导致开发返工 | 在待确认问题中列出，尽快与产品/业务方确认 |
| **技术依赖风险**：依赖多个外部系统（1.0、BI平台、智能问数、管理系统） | 外部系统不可用影响功能 | 确认各系统接口可用性和 SLA；设计降级方案 |
| **数据一致性风险**：收藏、点赞、浏览等数据需要多端同步 | 用户在不同设备看到的数据不一致 | 后端统一存储用户操作数据，前端实时同步 |
| **排期风险**：全平台所有页面范围较大 | 可能影响交付时间 | 按优先级分期交付：P0（总部看板+报表卡片+导航+搜索）→ P1（推荐系统+我的工作台+子页面）→ P2（定制+统计+移动端） |
| **1.0 到 2.0 过渡风险**：用户不习惯新版本 | 用户满意度下降 | 保留「返回旧版」入口，收集用户反馈持续优化 |

## 7. 非功能需求

| 维度 | 要求 |
|------|------|
| **性能** | 页面加载 < 2s，报表列表渲染 < 1s，搜索响应 < 1s |
| **安全** | 用户操作（收藏、点赞等）需要防刷机制；接口需要鉴权 |
| **稳定性** | 核心功能（导航、卡片展示、搜索）可用性 > 99.9% |
| **易用性** | 报表查找路径不超过 3 次点击；卡片操作有明确的视觉反馈 |
| **可维护性** | 报表卡片组件化；菜单和报表列表支持后端配置化 |
| **兼容性** | PC 端：Chrome、Edge、Firefox、Safari 最新两个版本；分辨率 1440×900 及以上 |
| **可观测性** | 接口耗时、异常需有日志；报表曝光、点击等关键行为需埋点 |
